# v1.5.0 规划：ccId 注册机制

## 背景

当前 `enter_headless_mode` 会接受任意 ccId，存在安全隐患：
- 任意会话可以使用任意 ccId 进入微信模式
- 可能导致消息路由混乱或冒充

**目标**：实现 ccId 注册验证机制，确保只有已注册的 ccId 才能进入微信模式。

---

## 功能需求

### 1. 注册流程

**场景 1**：无配置文件（首次使用）
```
1. 检查 .claude/wecom-aibot.json 不存在
2. 启动注册流程
3. 调用 register_ccId 注册
4. 注册成功后创建配置文件
5. 进入微信模式
```

**场景 2**：配置文件存在，但 ccId 未在 MCP 注册
```
1. 读取 wecom-aibot.json 获取 ccId
2. 调用 enter_headless_mode
3. MCP 返回 "ccId_not_registered"
4. 启动重新注册流程
5. 注册成功后更新配置文件
6. 进入微信模式
```

## 错误处理

| 错误码 | 说明 | 处理方式 |
|--------|------|---------|
| ccId_not_registered | ccId 未注册 | 启动注册流程 |
| ccId_occupied | ccId 被其他项目占用 | 提示用户选择新 ccId |
| registration_failed | 注册失败 | 检查 MCP 服务状态 |
